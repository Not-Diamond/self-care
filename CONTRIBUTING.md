# Contributing to Self-Care

Thank you for your interest in contributing to Self-Care!

## Bug Reports

[Open an issue](https://github.com/Not-Diamond/self-care/issues) with the **Bug report** template. A good bug report includes:

- Steps to reproduce the problem
- Expected vs actual behavior
- Trace file format used (OTEL JSON, Claude Code trace, etc.)
- Plugin version (`/self-care:help` to check)
- Claude Code version and OS

## Feature Requests

[Open an issue](https://github.com/Not-Diamond/self-care/issues) with the **Feature request** template. Tell us:

- What problem you're trying to solve
- Your current workaround (if any)
- How you'd expect the feature to work

## Pull Requests

We welcome PRs for bug fixes, documentation improvements, and new detection skills. Before starting significant work, please open an issue first to discuss the approach.

When submitting a PR:

- Keep changes focused — one fix or feature per PR
- Describe what the change does and why
- Include steps to test the change

## Security Vulnerabilities

Please see our [Security Policy](./SECURITY.md) for how to report vulnerabilities privately.

## Questions & Discussion

- **General questions**: Open a [discussion](https://github.com/Not-Diamond/self-care/discussions)
- **Support**: Reach out at [support@notdiamond.ai](mailto:support@notdiamond.ai)

Please read our [Code of Conduct](./CODE_OF_CONDUCT.md). We expect all participants to follow it.
